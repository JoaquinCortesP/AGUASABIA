# Documento de gestion y evidencias - AguaSabia

## 5.1 Configuracion del servidor y ambiente tecnico

Esta seccion registra la configuracion necesaria para ejecutar AguaSabia en ambiente de desarrollo, pruebas o despliegue controlado.

### Backend

El backend se ejecuta con Python y FastAPI.

Comandos base:

```powershell
cd C:\Users\Joaqu\OneDrive\Escritorio\AguaSabia\AGUASABIA\Proyecto\backend
.\.venv\Scripts\activate
pip install -r requirements.txt
```

Evidencias sugeridas:

- captura de entorno virtual activo;
- captura de dependencias instaladas;
- captura de `requirements.txt`;
- captura de `.env.example` o `.env` censurado.

### Base de datos

Se utiliza PostgreSQL. Para pruebas se recomienda una base separada:

```text
aguasabia_test
```

Comandos:

```powershell
python -m alembic upgrade head
python scripts\seed.py
```

Evidencias sugeridas:

- migraciones aplicadas;
- seed ejecutado;
- tablas principales disponibles.

### API

Comando:

```powershell
uvicorn app.main:app --reload
```

URLs:

```text
http://127.0.0.1:8000
http://127.0.0.1:8000/docs
```

Evidencias sugeridas:

- terminal con Uvicorn;
- Swagger abierto;
- Postman con rutas probadas.

## 5.2 Configuracion de ambiente de pruebas

El ambiente de pruebas debe permitir validar consultas territoriales por poligono sin afectar datos reales.

Elementos requeridos:

- backend local levantado;
- PostgreSQL con base `aguasabia_test`;
- seed ejecutado;
- Postman instalado;
- conexion a internet para Open-Meteo.
